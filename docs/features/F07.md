# Feature Overview

## Purpose

Identifies and persists discrete, trackable commitments (action items with assignees) and finalized agreements (decisions with confidence scores) from meeting transcripts. This transforms ephemeral spoken agreements into accountable, reviewable records.

## User Journey

1. During or after a meeting, the system analyzes the transcript for commitments and agreements.
2. In the post-meeting summary, the user sees extracted action items (e.g., "Alice will prepare the budget proposal by Friday") and decisions (e.g., "The team agreed to postpone the launch to Q2").
3. Action items display assignee information and can be tracked for completion status.
4. Decisions include a confidence score indicating how explicitly the agreement was stated.
5. These artifacts are visible in the meeting detail view and the Insights dashboard.
## Functional Behaviour

Action items and decisions are extracted as part of the summary generation process (F06) â€” the LLM prompt for summary generation explicitly requests `actionItems` (array of objects with `content`, `assignee`, `status`) and `decisions` (array of objects with `content`, `confidence`). These are embedded within the summary JSON response. Additionally, the system can extract these artifacts during live processing of individual transcript windows (`processLiveWindow`), associating each artifact with its `sourceWindowId` for provenance tracking. Extracted artifacts are persisted as independent entities in their own MongoDB collections (`ActionItem`, `Decision`).

## Processing

1. **Extract via LLM**: As part of summary generation, the LLM identifies action items and decisions from the transcript text.
2. **Parse Response**: Extract the `actionItems` and `decisions` arrays from the LLM JSON response.
3. **Persist**: Save each action item and decision as an independent entity in MongoDB, linked by `meetingId` and optionally by `sourceWindowId`.
4. **Serve**: Return the artifacts as part of the summary object and/or via dedicated query endpoints.
## Outputs

| Decisions | Array of `{ meetingId, content, confidence, sourceWindowId }` |

## Limitations

- Action item status tracking is limited; there is no workflow for marking items complete or assigning due dates beyond what the LLM extracts.
- Extraction quality depends entirely on the LLM; subtle or implicit commitments may be missed.
- No deduplication â€” the same action item spoken multiple times may be extracted as separate entries.
- Currently tightly coupled with the summary generation flow rather than being independently invocable.
