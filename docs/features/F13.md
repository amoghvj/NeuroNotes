# Feature Overview

| **Feature Name** | Meeting Scheduling via Voice |
| **Status** | Implemented |

## Purpose

Allows users to verbally schedule a follow-up meeting during a live session. This eliminates the need to context-switch to a calendar application while discussing future plans.

## User Journey

1. During a meeting, a participant says: "Hey NeuroNotes, schedule a meeting to review the design next Tuesday at 2 PM. Over."
2. The Command Interpreter detects the command and classifies it as `schedule_meeting`.
3. The LLM refines the raw text to extract a title ("Design Review") and a date/time ("next Tuesday at 2 PM").
4. A pending automation appears in the UI.
5. The user reviews the extracted title and time, makes any necessary adjustments, and clicks "Approve".
6. A new meeting record is created in the system with the specified title and time.
## Functional Behaviour

When the Command Interpreter (M3) classifies a voice command as `schedule_meeting`, it passes the payload to the Automation Orchestrator (M4). The Orchestrator calls the Intelligence Core (M5) with a prompt to extract scheduling parameters (`title`, `date`, `participants`) from the natural language text. The extracted parameters are stored in an `AutomationLog` with a `pending` status. Upon user approval via the UI (M12), the Orchestrator executes the intent by creating a new `Meeting` entity via M1's data layer, using the extracted title. The automation status is then updated to `completed`.

## Processing

1. **Intent Classification**: M3 identifies `schedule_meeting` intent.
2. **Parameter Extraction**: M4 sends command text to M5 â†’ LLM returns JSON with title and date.
3. **Pending Log**: M4 creates an `AutomationLog` with the extracted parameters.
4. **User Approval**: User approves the automation via UI.
5. **Execution**: M4 creates a new `Meeting` document with the extracted title and current time as startTime (NOTE: Current implementation flaw).
6. **Completion**: M4 marks the automation as `completed`.

## Limitations

- The current implementation creates a new meeting immediately in the `live` state instead of scheduling it for a future date (the extracted date is ignored during creation).
- No calendar integration â€” the meeting is only scheduled within NeuroNotes, not added to Google Calendar or Outlook.
- Extraction relies entirely on the LLM; relative dates ("next Tuesday") might be resolved incorrectly without explicit reference dates in the prompt.
- Fails silently if the LLM cannot extract a title.
