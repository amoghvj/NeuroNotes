## Purpose

Provides a text-based AI assistant that users can query about the content of a meeting. Rather than re-reading the entire transcript, users can ask natural language questions ("What did Alice say about the budget?") or use slash commands (/summary, /actions) to get instant, context-grounded answers. This makes meeting intelligence interactive and accessible.

## User Journey

1. The user opens the Chat panel in the Live Meeting or Meeting Detail view.
2. They type a question or slash command (e.g., `/summary` or "What decisions were made?").
3. The system assembles transcript context, constructs a grounded prompt, and sends it to the LLM.
4. Within seconds, the AI response appears in the chat panel, formatted as markdown.
5. Response metadata is shown alongside the answer: analysis window, speakers involved, and confidence score.
6. The user can continue asking follow-up questions in the same session.
## Functional Behaviour

The chat handler receives a meeting ID and query string. It fetches all MinuteWindow documents for the meeting and concatenates them into a single context string. If the query starts with a slash command (`/summary`, `/actions`, `/decisions`, `/insights`), it is expanded into a more detailed, optimized prompt. The system prompt explicitly instructs the LLM to answer only based on the provided transcript context and to refuse questions outside scope. The grounded prompt is sent to M5. The response text is returned alongside metadata including: the analysis time window, list of unique speakers in the context, and a confidence score. In DEMO_MODE, hardcoded fallback responses are returned for each slash command without calling the LLM.

1. **Receive**: Accept the query and meeting ID from the frontend.
2. **Fetch Context**: Retrieve all MinuteWindow documents â†’ Concatenate into a single transcript string.
3. **Slash Command Detection**: If query starts with `/`, expand it into a detailed prompt (e.g., `/actions` â†’ "List all action items mentioned with assignees and deadlines").
4. **Compose Prompt**: Build the system prompt with grounding instructions + transcript context + user query.

## Data Created Or Modified

No persistent entities are created. Chat history is ephemeral on the frontend and not persisted to the backend.

## Limitations

- The entire transcript is fetched and concatenated for every query â€” no windowing or bounded context retrieval. This will fail for long meetings exceeding LLM token limits.
- Chat history is not persisted; refreshing the page loses all previous Q&A.
- No multi-turn memory â€” each query is independent; follow-up questions like "Tell me more about that" lack context of the previous answer.
- Slash command fallback responses in DEMO_MODE are hardcoded inside `LLMService` rather than being owned by the chat module.
- No semantic retrieval (RAG) â€” context selection is "all or nothing" rather than relevance-based.
