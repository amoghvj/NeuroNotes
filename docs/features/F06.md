## Purpose

Automatically produces a comprehensive, structured summary of an entire meeting immediately after it ends. This eliminates the need for manual note-taking and ensures all participants have access to a consistent, AI-generated record of what was discussed, decided, and assigned.

## User Journey

1. The user clicks "End Meeting" in the Live Meeting view.
2. The meeting status transitions to `completed` and the UI confirms the meeting has ended.
3. In the background, the system fetches the full transcript and sends it to the LLM for analysis.
4. Within seconds, a structured summary appears on the meeting detail page containing key points, decisions, action items, opportunities, risks, eligibility criteria, and open questions.
5. This summary is also used as the basis for the email summary automation (F12).
## Functional Behaviour

When `endMeeting` is called, post-meeting processing runs asynchronously (via `setImmediate`). The system fetches all MinuteWindow documents for the meeting and concatenates them into a single transcript string. This string is sent to the LLM via a hardcoded prompt that instructs the model to return a strictly formatted JSON object containing: keyPoints (array of strings), decisions (array of objects), actionItems (array of objects with assignee and dueDate), opportunities, risks, eligibility, and questions. The LLM response is cleaned (markdown code block markers stripped) and parsed as JSON. The resulting summary object is saved directly to the `Meeting.summary` field. In DEMO_MODE, a hardcoded mock summary is returned instead of calling the LLM.

## Processing

1. **Trigger**: Initiated asynchronously after `endMeeting` updates the meeting status.
2. **Fetch Transcript**: Retrieve all MinuteWindow documents for the meeting â†’ Concatenate into a single string.
3. **Compose Prompt**: Build the LLM prompt with strict JSON output instructions and the transcript as context.
4. **Call LLM**: Send prompt to M5 (Intelligence Core) â†’ Receive raw response string.
5. **Parse Response**: Strip markdown formatting â†’ Parse as JSON â†’ Validate structure.
6. **Persist**: Save the parsed summary object to `Meeting.summary`.
7. **Trigger Email**: Create a pending `email_summary` AutomationLog entry with the summary data.

## Limitations

- The entire transcript is sent in a single LLM call â€” no map-reduce summarization for very long meetings that may exceed token limits.
- Summary generation prompt and logic is currently embedded in `LLMService` rather than isolated in the Insight Generator module.
- If the LLM returns malformed JSON, the summary silently fails and the meeting has no summary.
- Summary quality is entirely dependent on the LLM's ability to follow the prompt; no fallback or quality scoring mechanism.
- In DEMO_MODE, a static mock summary is returned regardless of actual transcript content.
