## Purpose

Automatically distributes a professionally formatted email containing the meeting summary to relevant stakeholders after a meeting ends. This ensures all team members â€” including those who may not use the NeuroNotes UI â€” receive actionable meeting output in their inbox without manual effort.

## User Journey

1. A meeting ends, triggering post-meeting processing.
2. The system generates the meeting summary (F06) and creates a pending `email_summary` automation.
3. The user sees the pending email automation in the UI with the resolved recipient list and summary preview.
4. The user reviews the recipients, optionally edits them, and clicks "Approve."
5. The system composes an HTML email body from the summary data and dispatches it via the n8n webhook.
6. n8n handles the actual email delivery to the resolved recipients.
7. The automation status updates to `completed`, and the external execution ID from n8n is recorded.
## Functional Behaviour

When the `email_summary` automation is approved, the orchestrator resolves the recipient list. If the user provided `selectedRecipients` on the meeting, those are used. Otherwise, the system falls back to querying the Workspace module (M11) for the default member list. The recipient emails are joined into a comma-separated string. The summary data (key points, decisions, action items) is formatted into an HTML email body using inline CSS (required by email clients). The composed payload â€” recipients string, subject line, HTML body â€” is sent via HTTP POST to the configured `N8N_WEBHOOK_URL`. The n8n workflow handles the actual SMTP delivery. The response from n8n (including an execution ID) is recorded on the AutomationLog for auditability.

## Processing

1. **Resolve Recipients**: Check for user-edited recipients â†’ Fall back to meeting's selectedRecipients â†’ Fall back to workspace members via M11.
2. **Format Email Body**: Iterate over key points, decisions, and action items â†’ Build HTML lists with inline CSS.
3. **Compose Payload**: Assemble `{ to: recipientsString, subject: "Meeting Summary: {title}", html: emailBody }`.
4. **Dispatch Webhook**: HTTP POST to `N8N_WEBHOOK_URL` with the payload.
5. **Record Result**: Save n8n response and external execution ID to the AutomationLog.
6. **Update Status**: Transition AutomationLog status to `completed` or `failed`.

## Limitations

- Email composition logic and recipient resolution are scattered across `AutomationService` and `meetingController` rather than being in a dedicated Communication module.
- No retry mechanism for failed webhook dispatches.
- HTML email templates are built via string interpolation â€” no templating engine, making complex layouts fragile.
- No direct email integration; entirely dependent on n8n as an intermediary.
- No notification preferences â€” all workspace members receive the email with no opt-out mechanism.
- No delivery tracking â€” the system records the n8n execution ID but does not verify actual email delivery.
