## Purpose

Enables participants to create data visualizations (bar charts, line graphs, pie charts, timelines) directly from spoken data during a meeting. This bridges the gap between verbal discussion of numbers and visual representation, allowing teams to instantly see charts rendered from conversational data.

## User Journey

1. During a live meeting, a participant verbally describes data: "Hey NeuroNotes, create a chart showing Q1 revenue at 2.3 million, Q2 at 2.8 million, Q3 at 3.1 million, and Q4 at 3.5 million. Over."
2. The Command Interpreter detects the voice command and classifies the intent as `create_visualization`.
3. The text block containing the data is passed to the Visualization Pipeline.
4. The LLM extracts structured chart data (labels: ["Q1","Q2","Q3","Q4"], values: [2.3, 2.8, 3.1, 3.5]) and suggests a chart type (bar).
5. The configuration is validated (label/value count match, all values are numeric).
6. The chart appears live in the VisualIntelligence section of the Live Meeting view, rendered by Chart.js.
## Functional Behaviour

The Visualization Pipeline receives the text content block (isolated by the Command Interpreter) and constructs an LLM prompt requesting strict JSON output with fields: `type` (bar/line/pie/timeline), `title`, `description`, `data.labels` (array of strings), and `data.values` (array of numbers). The LLM response is cleaned of markdown formatting and parsed as JSON. The `VisualEngine.validateCandidate` function then checks: required fields exist, `labels` and `values` are arrays of equal length, all values are numeric. If validation passes, the artifact is persisted as a `VisualArtifact` document in MongoDB, linked to the meeting ID and optionally to a source window ID. The frontend polls for new visual artifacts and renders them using Chart.js. There is also a legacy trigger path via `VisualizationTriggerService` that independently scans for "start chart / end chart" phrases, creating a dual entry point.

## Limitations

- Supported chart types are limited to bar, line, pie, and timeline.
- Dual trigger entry points (Command Interpreter + legacy VisualizationTriggerService) create race conditions and potential double-generation.
- No interactive editing â€” users cannot modify a chart after creation (e.g., "Change to pie chart").
- Only single-series data supported â€” no grouped bar charts or multi-line graphs.
- If the LLM hallucinates data that passes validation (correct format but wrong numbers), there is no detection mechanism.
- The `VisualizationTriggerService` still contains its own trigger detection logic, violating ownership boundaries.
